# THE GREAT WORK
## Working Draft V0.1

### Purpose

This document exists to prevent strategic drift.

It defines:

- what is being built
- why it is being built
- in what order it should be built
- how success is measured

This document is expected to evolve.

---

# CORE THESIS

The product is not the chair.

The product is the factory that builds the chair.

The goal is not autonomous action.

The goal is autonomous momentum.

The goal is not replacing humans.

The goal is increasing the ability of humans and Aeyes to coordinate, execute, remember, and continue moving.

---

# BOOK I — ESCAPE VELOCITY

## Objective

Remove Ben as middleware.

Current State:

Ben acts as:

- message bus
- hall monitor
- historian
- work router
- context carrier
- approval button

Target State:

Ben acts as:

- operator
- destination setter
- exception handler

Success Condition:

The machine continues moving while Ben is unavailable.

# BOOK IV — WERKLES

## Purpose

Help projects maintain momentum.

Domain agnostic.

Applicable to:

- businesses
- factories
- teams

- communities
- families
- startups

---

## Core Loop

Understand current state.

Determine next useful action.

Execute.

Observe outcome.

Update understanding.

Repeat.

---

## Design Principle

Reduce friction between intention and execution.

Every feature should improve:

- awareness
- coordination
- execution
- memory
- momentum

Features that improve none of these are outside the critical path.
